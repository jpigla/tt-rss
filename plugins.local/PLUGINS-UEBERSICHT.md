# TT-RSS Custom Plugins — Übersicht

> 54 Plugins in `plugins.local/`, organisiert nach Funktionsbereich.
> Diese Übersicht wird bei Plugin-Änderungen aktuell gehalten.

---

## Phase 1 — Lese-Erlebnis


| Plugin                | Beschreibung                                                                                                                                                                                                                                                                                                                                        |
| --------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **reading_time**      | Berechnet die geschätzte Lesezeit pro Artikel anhand der Wortanzahl (inkl. CJK-Zeichen) und zeigt sie als Badge oberhalb des Textes an. Die Lesegeschwindigkeit (WPM) ist pro Benutzer konfigurierbar (Standard: 200 WPM).                                                                                                                          |
| **boosted_feeds**     | Ermöglicht kürzere Aktualisierungsintervalle für ausgewählte Feeds. Im Feed-Editor kann pro Feed ein Boost aktiviert werden, der das Update-Intervall auf einen konfigurierbaren Wert setzt (Standard: 5 Min.). Nutzt die bestehende `update_interval`-Spalte, sodass der Update-Daemon die Feeds automatisch häufiger abruft.                      |
| **keyword_spotlight** | Hebt definierte Keywords in Artikeln farblich hervor, damit relevante Stellen beim Scannen schneller auffallen. Unterstützt bis zu 5 Farbgruppen mit je eigener Farbe (Color-Picker). Die Hervorhebung erfolgt beim Rendern via DOMDocument/XPath, sodass neue Keywords sofort auf alle Artikel wirken.                                             |
| **filter_log**        | Protokolliert jede Filter-Aktion (Treffer, Regeln, Aktionen) in einer eigenen DB-Tabelle und zeigt sie als durchsuchbare Detailtabelle in den Einstellungen an. Enthält Tagesstatistiken und einen „Log leeren"-Button. Automatische Bereinigung nach 30 Tagen bzw. 1000 Einträgen via Housekeeping.                                                |
| **read_later**        | Später-Lesen-Funktion über ein dediziertes Label (`📌 Später lesen`). Artikel können per Artikel-Button oder Hotkey `l` zur Leseliste hinzugefügt/entfernt werden. Durch die Label-basierte Implementierung integriert sich die Funktion nahtlos in Sidebar-Zähler, Suche, Filter und API.                                                          |
| **af_fulltext**       | Extrahiert den vollständigen Artikeltext von der Quell-Website und ersetzt den Feed-Ausschnitt. Nutzt eine 3-stufige Heuristik: semantische HTML5-Tags, Textdichte-Analyse und Body-Fallback. Der Volltext wird in `cached_content` gecacht und durch den Standard-Sanitizer bereinigt. Auch von anderen Plugins über `HOOK_GET_FULL_TEXT` nutzbar. |
| **enhanced_tags**     | Erweitert das TT-RSS-Tagging um einen benutzerfreundlichen Dialog mit Autocomplete und Vorschlags-Chips (Top-30 Tags). Zeigt Tag-Chips direkt in Artikeln an und bietet eine Tag-Cloud in den Einstellungen mit Größen nach Häufigkeit. Nutzt die bestehende `ttrss_tags`-Tabelle ohne eigenes Schema.                                              |
| **dedup_filter**      | Erkennt und unterdrückt doppelte Artikel beim Feed-Import anhand von Titel-Ähnlichkeit (PostgreSQL `pg_trgm`) und URL-Vergleich. Duplikate werden automatisch als gelesen markiert. Konfigurierbar pro Feed im Feed-Editor mit einstellbarem Ähnlichkeits-Schwellwert.                                                                              |


## Phase 2 — Quellenaufnahme


| Plugin               | Beschreibung                                                                                                                                                                                                                                                                                             |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **web_feeds**        | Ermöglicht das Abonnieren von Webseiten ohne RSS-Feed mittels CSS-Selektoren. Im Feed-Editor werden Selektoren für Container, Titel, Link und Inhalt definiert; das Plugin scrapt die Seite und generiert daraus einen synthetischen RSS-Feed. Enthält eine Vorschau-Funktion zum Testen der Selektoren. |
| **social_feeds**     | Wandelt Social-Media-URLs (Reddit, YouTube, GitHub, Mastodon) beim Abonnieren automatisch in die zugehörigen RSS-Feed-URLs um. Löst YouTube-Handles und Mastodon-Profilnamen zu Feed-URLs auf. Konfigurierbar in den Einstellungen, welche Plattformen aktiv sein sollen.                                |
| **news_search_feed** | Erlaubt das Abonnieren von Google-News-Suchanfragen als RSS-Feeds. Suchbegriffe werden in Google-News-RSS-URLs konvertiert, mit konfigurierbarer Sprach- und Regionseinstellung. Die Feeds werden dann regulär wie jeder andere Feed aktualisiert.                                                       |
| **track_changes**    | Überwacht beliebige Webseiten auf inhaltliche Änderungen und erzeugt bei Änderungen automatisch Feed-Einträge. Unterstützt optionale CSS-Selektoren zur Eingrenzung des überwachten Bereichs. Prüfintervall und überwachte URLs werden in den Einstellungen verwaltet.                                   |
| **monitoring_feeds** | Keyword-basiertes Artikel-Monitoring: Definierte Suchbegriff-Sets werden bei jedem Feed-Import gegen neue Artikel geprüft. Treffer werden automatisch mit `monitor:<Name>`-Tags versehen. Mehrere Monitor-Profile mit unterschiedlichen Keywords sind pro Benutzer konfigurierbar.                       |
| **podcast_player**   | Erweiterter HTML5-Audio-Player für Podcast-Enclosures mit Geschwindigkeitssteuerung (0,5×–2×), 15s-Skip-Buttons und Fortschrittsspeicherung. Der Wiedergabefortschritt wird serverseitig gespeichert und beim erneuten Öffnen wiederhergestellt. Ersetzt die Standard-Enclosure-Darstellung.             |
| **save_page**        | Speichert beliebige Webseiten per URL als Artikel in den „Gespeicherte Artikel"-Feed. Extrahiert automatisch Titel und Hauptinhalt der Seite. Enthält SSRF-Schutz (keine internen IPs) und Inhaltsbereinigung via Sanitizer. Auch per Bookmarklet und Browser-Extension nutzbar.                         |
| **youtube_sync**     | Importiert YouTube-Abonnements aus Google-Takeout-OPML-Dateien und legt sie als Feeds in einer konfigurierbaren Kategorie an. Bettet YouTube-Videos direkt in die Artikel-Ansicht ein (responsive iFrame). Erkennt YouTube-URLs automatisch und ersetzt die Standard-Darstellung.                        |
| **feed_autodetect**  | Erweiterte Feed-Erkennung beim Abonnieren von URLs: Durchsucht HTML-Seiten nach `<link>`-Tags und prüft 14 gängige Feed-Pfade (`/feed`, `/rss`, `/atom.xml` etc.). Korrigiert HTML5-Boolean-Attribute in der XML-Verarbeitung. Zeigt gefundene Feeds zur Auswahl an.                                     |


## Phase 3 — Automatisierung


| Plugin                | Beschreibung                                                                                                                                                                                                                                                                                |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **webhooks**          | Sendet HMAC-signierte HTTP-Webhooks (JSON) bei Filter-Auslösungen an konfigurierte HTTPS-Endpunkte. Mehrere Webhook-URLs mit individuellen Secrets sind möglich. Enthält einen Test-Button zum Überprüfen der Konnektivität. Nur HTTPS-Endpunkte werden akzeptiert.                         |
| **push_notify**       | Push-Benachrichtigungen bei Filter-Auslösungen über ntfy.sh, Gotify oder Pushover. Pro Dienst werden Endpunkt/Token konfiguriert; bei Filter-Treffern wird eine Benachrichtigung mit Artikeltitel und Feed-Name gesendet. Mehrere Dienste können parallel aktiv sein.                       |
| **output_feeds**      | Generiert öffentliche RSS-Feeds aus Ordnern, Labels oder Tags. Jeder Output-Feed erhält einen individuellen Access-Key für die URL. Konfigurierbar sind Quelle, Artikelanzahl und Titel. Die Feeds sind ohne Login über die generierte URL abrufbar.                                        |
| **save_to**           | Export von Artikeln zu externen Diensten: Wallabag, Pocket, Linkding und Instapaper. Pro Dienst werden API-Credentials in den Einstellungen konfiguriert. Der Export erfolgt per Artikel-Button mit einem Klick. Unterstützt Tags/Labels-Übernahme wo die Ziel-API dies erlaubt.            |
| **pdf_export**        | Exportiert Artikel als druckoptimierte HTML-Seite, die über den Browser-Druckdialog als PDF gespeichert werden kann. Die Export-Ansicht enthält Titel, Autor, Datum, Quell-URL und bereinigten Inhalt mit optimierter Typographie. Öffentlich zugänglich über einen generierten Link.       |
| **enhanced_search**   | Gespeicherte Suchen mit Quick-Access: Komplexe Suchanfragen können mit Namen verspeichert und per Toolbar-Button schnell aufgerufen werden. Unterstützt erweiterte Filter nach Feed, Kategorie und Datumsbereich. Die gespeicherten Suchen erscheinen als Dropdown in der Toolbar.          |
| **enhanced_filters**  | Erweiterte Filter-Regeln über die Standard-Filter hinaus: Zusätzliche Bedingungen wie Artikellänge, Regex-Muster und Score-Schwellwerte. Zusätzliche Aktionen wie Auto-Tagging und Archivierung. Konfiguration über einen eigenen Prefs-Tab.                                                |
| **bulk_actions**      | Massenoperationen auf mehrere ausgewählte Artikel gleichzeitig: Tags zuweisen, Labels setzen, als gelesen markieren. Integriert sich in das Headline-Toolbar-Kontextmenü über `HOOK_HEADLINE_TOOLBAR_SELECT_MENU_ITEM2`.                                                                    |
| **browser_extension** | CORS-fähiger API-Endpunkt für Browser-Erweiterungen und Bookmarklets. Bietet Funktionen zum Speichern von Webseiten als Artikel, Annotationen erstellen, Labels verwalten und Read-Later-Status umschalten. Authentifizierung über Session oder API-Token mit konfigurierbarem CORS-Origin. |


## Phase 4 — KI & Intelligence


| Plugin         | Beschreibung                                                                                                                                                                                                                                                                                                                                                                     |
| -------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ai_core**    | Zentrale LLM-Abstraktionsschicht für alle KI-Plugins. Unterstützt vier Provider: Ollama (lokal, Standard), OpenAI, Anthropic und Custom-Endpunkte (OpenAI-kompatibel). Bietet eine statische API (`Ai_Core::complete()`) mit einheitlichem Interface und einen Verbindungstest-Button in den Einstellungen. Alle KI-Plugins setzen dieses Plugin voraus.                         |
| **ai_summary** | Generiert on-demand KI-Zusammenfassungen für Artikel in drei Modi: Kurz (1–2 Sätze), Stichpunkte (3–5 Bullet Points) und Ausführlich (3–5 Sätze mit Kontext). Ergebnisse werden in einer eigenen DB-Tabelle gecacht, sodass wiederholte Aufrufe sofort antworten. Die Zusammenfassung erscheint als farbige Box oberhalb des Artikels mit Modus-Umschaltung. Benötigt `ai_core`. |
| **ai_prompts** | Ermöglicht das Anwenden benutzerdefinierter KI-Prompts auf Artikel. Prompts werden in den Einstellungen erstellt und verwaltet; per Artikel-Button wird ein Prompt ausgewählt und das LLM-Ergebnis angezeigt. Nützlich für individuelle Analysen, Extraktionen oder Umformulierungen. Benötigt `ai_core`.                                                                        |
| **ai_chat**    | Chat-Interface zum Stellen von Fragen an einzelne Artikel. Der Artikelinhalt wird als Kontext an das LLM übergeben, sodass gezielte Fragen zum Inhalt beantwortet werden können. Dialog-basierte UI mit Chatverlauf. Benötigt `ai_core`.                                                                                                                                         |
| **translate**  | Artikelübersetzung mit drei Backend-Optionen: LibreTranslate (Self-Hosting), DeepL (API-Key) und KI-basiert (über `ai_core`). Die Zielsprache ist konfigurierbar; Übersetzungen werden gecacht. Per Artikel-Button wird der Inhalt in-place übersetzt oder als Overlay angezeigt.                                                                                                |
| **tts**        | Text-to-Speech: Liest Artikel über die Browser Web Speech API oder einen Edge-TTS-Server vor. Unterstützt Sprachwahl, Geschwindigkeitssteuerung und Streaming-Audio. Bei Edge-TTS wird das Audio serverseitig generiert und als Stream ausgeliefert. Konfigurierbar in den Einstellungen.                                                                                        |
| **ai_tags**    | KI-gestützte Tag-Vorschläge für Artikel mit optionalem Auto-Tagging beim Feed-Import. Nutzt TF-IDF und Stopwort-Filterung zur Keyword-Extraktion, die vom LLM zu sinnvollen Tags verdichtet werden. Tags können manuell übernommen oder automatisch zugewiesen werden. Benötigt `ai_core`.                                                                                       |
| **magic_sort** | Relevanz-basierte Artikelsortierung anhand von Feed-Engagement-Scores. Berechnet pro Feed einen Score aus Lese-, Stern- und Interaktionshistorie. Artikel aus häufiger gelesenen Feeds erscheinen weiter oben. Scores werden via Housekeeping periodisch aktualisiert.                                                                                                           |
| **ai_reports** | Generiert KI-gestützte Intelligence Reports aus mehreren ausgewählten Artikeln. Fasst Themencluster zusammen, identifiziert Gemeinsamkeiten und Trends. Ergebnisse werden als strukturierte Berichte dargestellt. Aufruf über einen Toolbar-Button. Benötigt `ai_core`.                                                                                                          |


## Phase 5 — Kollaboration & UI


| Plugin                | Beschreibung                                                                                                                                                                                                                                                                                                          |
| --------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **annotations**       | Text-Highlighting und Annotationen direkt in Artikeln. Textpassagen können farblich markiert und mit Notizen versehen werden. Annotationen werden in einer eigenen DB-Tabelle gespeichert und beim Rendern wieder eingeblendet. Unterstützt Suche über annotierte Inhalte und Export. Kontextmenü-basierte Bedienung. |
| **reading_progress**  | Speichert und zeigt den Lesefortschritt pro Artikel als Fortschrittsbalken am Artikel-Header an. Der Fortschritt wird beim Scrollen automatisch erfasst (0–100 %) und serverseitig gespeichert. Beim erneuten Öffnen scrollt der Artikel zur letzten Position.                                                        |
| **resizable_sidebar** | Ermöglicht das Ändern der Seitenleistenbreite per Drag & Drop. Die gewählte Breite wird im localStorage persistiert (Bereich: 150–600 px). Rein clientseitige Implementierung ohne Backend-Hooks.                                                                                                                     |
| **dashboards**        | Konfigurierbare Dashboards mit Widgets für Statistiken und Übersichten. Verfügbare Widgets: Ungelesene Artikel, kürzlich gelesene Artikel, Labels, Lese-Statistiken. Mehrere Dashboards pro Benutzer möglich, mit Drag & Drop-Anordnung der Widgets. Zugang über einen Toolbar-Button.                                |
| **edit_metadata**     | Ermöglicht das nachträgliche Bearbeiten von Artikelmetadaten (Titel und Autor). Änderungen werden pro Artikel in der Datenbank gespeichert und überschreiben die Feed-Werte. Nützlich bei fehlerhaften oder fehlenden Metadaten aus dem Feed.                                                                         |
| **newsletter_feeds**  | Ruft E-Mail-Newsletter per IMAP ab und legt sie als echte Feed-Einträge an. Unterstützt mehrere IMAP-Konten mit konfigurierbaren Ordnern und Absender-Filtern. HTML-Styling der E-Mails wird erhalten; Absender-Banner werden injiziert. Periodischer Abruf via Housekeeping.                                         |
| **team_spaces**       | Team-Arbeitsbereiche zum Teilen und Diskutieren von Artikeln. Teams können erstellt, Mitglieder eingeladen und Artikel mit dem Team geteilt werden. Geteilte Artikel erscheinen in einem Team-Feed. Verwaltung über Prefs-Tab und Artikel-Button.                                                                     |
| **file_uploads**      | Upload von TXT-, HTML-, PDF- und DOCX-Dateien, die als veröffentlichte Artikel angelegt werden. Text wird automatisch extrahiert und als Artikelinhalt gespeichert. Upload über einen Toolbar-Button mit Drag & Drop-Unterstützung.                                                                                   |


## Phase 6 — Wissensarbeit, Analytics & Discovery


| Plugin             | Beschreibung                                                                                                                                                                                                                                                                                                                                                               |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **save_to_pkm**    | Export von Artikeln und Highlights an PKM-Tools: Readwise (Artikel + Highlights via Reader API), Notion (Seite in konfigurierter Datenbank mit Metadaten und Callout-Blöcken) und Obsidian (Markdown-Note per URI-Link oder direkt als Datei im Vault). Integriert automatisch Highlights aus dem Annotations-Plugin. Konfigurierbares YAML-Frontmatter für Obsidian.      |
| **reading_stats**  | Lesegewohnheiten tracken und als Analytics-Dashboard visualisieren. Automatisches Tracking bei >50 % Sichtbarkeit; Lesezeit-Erfassung in 30-Sekunden-Intervallen. Dashboard mit Übersichtskarten, Lese-Streak, Tages-Balkendiagramm (30 Tage), GitHub-Style Contribution-Heatmap (52 Wochen), Tageszeit-Verteilung und Top-10-Feeds. Konfigurierbare Aufbewahrungsdauer.   |
| **feed_health**    | Überwacht die Gesundheit aller Feeds: Erfolg/Fehler-Rate, Response-Zeiten und Staleness. Health-Dashboard mit Ampel-Übersicht (Gesund/Inaktiv/Fehler), Fortschrittsbalken, Problem-Feed-Liste und Fehler-Log. Pro Feed wird ein Health-Score (0–100 %) berechnet und im Feed-Editor angezeigt. Konfigurierbarer Staleness-Schwellwert (Standard: 14 Tage).                 |
| **digest_view**    | Konfigurierbarer Daily/Weekly Digest: Verdichtete Zusammenfassungen aus ausgewählten Feeds und Kategorien. Mehrere Digest-Profile pro Benutzer mit Häufigkeit (täglich/wöchentlich), Uhrzeitsteuerung und Filtern (Feeds, Kategorien, Mindest-Score, Max-Artikel). In-App Viewer mit Kategorie-Gruppierung, Digest-Archiv zum Durchblättern und optionalem E-Mail-Versand. |
| **feed_discovery** | Trending Topics und Feed-Empfehlungen. Automatische Keyword-Analyse über alle Feeds via PostgreSQL `tsvector_combined` mit Rising-Topics-Erkennung (aktuelle vs. vorherige Periode). Tag-Cloud-Visualisierung mit Beispielartikeln. Feed-Vorschläge basierend auf beliebten Feeds anderer Nutzer mit Ein-Klick-Abo.                                                        |


## Zusätzliche Plugins


| Plugin               | Beschreibung                                                                                                                                                                                                                                                                                                                                                           |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **reader_mode**      | Dedizierter Lesemodus mit anpassbarer Typographie (Schriftgröße, Zeilenhöhe, Maximalbreite). Öffnet Artikel in einem Drei-Spalten-Layout mit Inhaltsverzeichnis (TOC) und Metadaten-Sidebar. Navigation zwischen Artikeln per Vor-/Zurück-Buttons. Einstellungen werden pro Benutzer gespeichert.                                                                      |
| **compact_images**   | Kompakte Bilddarstellung in der Artikelansicht: Das erste Bild wird als Thumbnail angezeigt, die Original-URL erscheint unter dem Artikeltitel. Konfigurierbar in den Einstellungen mit Optionen für Bildgröße und URL-Anzeige.                                                                                                                                        |
| **parser_rules**     | Domain-spezifische Extraktionsregeln, die aus Leser-Selektionen gelernt werden. Erzeugt XPath/CSS-Selektoren aus markiertem Text, optional LLM-optimiert. Regeln werden automatisch deaktiviert bei niedriger Konfidenz. Verwaltung über Prefs-Tab mit Test- und Regenerierungs-Funktion. Wird von `HOOK_GET_FULL_TEXT` mit Priorität 25 vor `af_fulltext` ausgeführt. |
| **mobile_pwa**       | Progressive-Web-App-Optimierungen für mobile Nutzung: Touch-UI-Anpassungen, Safe-Area-Unterstützung für Geräte mit Notch, Swipe-Navigation zwischen Artikeln. Rein clientseitige Implementierung über CSS und JavaScript ohne Backend-Hooks.                                                                                                                           |
| **readwise_theme**   | Readwise-Reader-inspiriertes Design-System mit dunklem Theme, warmen Akzentfarben und optimierter Lesetypographie. Konfigurierbare Schriftoptionen und Metadaten-Anzeige in der Artikelansicht. Integriert sich über Render-Hooks in die Artikeldarstellung.                                                                                                           |
| **archive_articles** | Artikel archivieren und dearchivieren mit Erhalt der Feed-Icons und Metadaten. Archivierte Artikel werden in einem separaten Bereich verwaltet. Konfiguration und Archiv-Verwaltung über den Prefs-Tab.                                                                                                                                                                |
